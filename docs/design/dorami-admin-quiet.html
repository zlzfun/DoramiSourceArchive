设计推导:机房仪表墙(ops wall)

三个观测域(用户/内容/AI)数据形态迥异,segmented 保留是正确工具(与接入页退役 segmented 相反, 互为参照);但时间窗控件上移页头,一处控制全页窗口(内容页的收藏/订阅为累计口径,注记豁免)。 KPI 总账条沿用台账 28/700 tabular 刻度,但为被动读数、数字全 ink——语义色只留异常, 不再逐格彩色(原实现的彩色数字是「AI 味」信号)。

图表纪律(dataviz 校验记录)

  • 分类调色板过六项校验(validate_palette.js):亮面 6 槽白面 PASS;暗面按 L 0.48–0.67 带 同色相族降档(teal→#0d9488、amber→#d97706、sky→#0284c7、indigo→#8b84f0),#20252f 面全项 PASS; 「其它」槽恒为中性灰。对比 WARN 由图例+直标+tooltip 承接。
  • 色随实体不随排位:维度切换(按用途/按用户)重排系列,颜色跟实体走;固定序取色,不循环生成第 7 色—— 超出并入「其它」。
  • 堆叠面积段间 2px 表面缝;横条 4px 圆头数据端、组内 2px 缝、数值 mono 直标;网格 hairline 隐退、 mono 轴标;≥2 系列必有图例 + 末端选择性直标(不逐点标数);hover 十字线 + tooltip 默认配备。
  • 单轴纪律:调用与 tokens 两个量纲 = 两张图,不做双轴。

结构与范式复用

  • 账户表 = ledger-table 语法(46px 行、hover soft、选中竖条),行级管理动作收进 hover rowacts (AI 开关/重置密码/停用/删除,危险动作红 hover)——常态是一张安静的读数表。
  • 用户活动抽屉 = ledger-drawer 语法:登录卡(details 展开近 5 次)、三 tiles、各源阅读/收藏双系列条、 每日 AI 用量堆叠面积(与主看板同一图表引擎)。
  • AI 总闸 = 一条开关板(状态灯 + switch + hairline 分隔 + 行内模型配置),不再是独立大卡; 「测试连通/保存」quiet 级。
