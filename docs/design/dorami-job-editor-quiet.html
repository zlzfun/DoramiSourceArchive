设计推导:班次编排单(dispatch sheet)

任务 = 一组节点 + 一个时刻,编辑器按此拆三段:头 = 时刻(名称/启用/整体 cron + 人话回显与 「距下次」走字——把时刻表页的仪器读数带进编辑现场);体 = 目录 → 编排单(左侧勾选,右侧即时 出现该节点的班次行:cron 覆盖 + schema 参数;有单独时刻的行挂 wash「单独时刻」章);脚 = 唯一 CTA。

与现实现的差异(决策点)

  • 目录按板块分组(厂商官方/科技媒体/…,沿用节点页分类学),不再是无分组长列表;已编入行 wash 底;
  • cron 人话回显 + 距下次预览:前端粗解析常见五段式,解析不了则只显示原式(诚实降级);
  • 编排单行 hover 才出移除钮(rowacts 语法);参数字段紧凑行内(28px input),布尔项原生 checkbox;
  • 「启用任务」从脚部 checkbox 升到头部 switch(它是任务级状态,不是表单杂项);
  • 脚部注记「保存后时刻表立即更新」呼应调度台;取消/保存右对齐,保存唯一 primary。

复用范式:well 输入/ledger-switch/wash 状态标记/rowacts hover/r-overlay 模态 + sh-3(全站唯一强高程场景)。 亮暗均走 token,右下月亮切换。本文件:docs/design/dorami-job-editor-quiet.html
