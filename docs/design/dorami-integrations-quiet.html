设计推导:出口面板(patch bay)

接入页回答一个问题:「下游怎么把内容取走?」——所以页面就是通道清单:每个通道一张卡, 卡内恒定三件事:状态章 → 端点/凭据 → 接入片段。MCP 是能力最富的通道(五个工具), 占左侧大卡;聚合接口与技能包纵排右列。

两项退役决策(需拍板)

  • 「日报生成 / Agent 接入」segmented 退役:与运行页同理,分区就地可见优于藏在视图切换后; 日报工坊收敛为一张三段卡(配置/生成/近期),不再需要整页切换。
  • integration-hero 大渐变面撤除:P4 曾标注「接入页唯一品牌陈述面,是否降级留给布局波」—— 本设计的答案是撤:静默仪器里品牌感归登录页,接入页的内容就是端点与凭据本身; 页头右侧只留一枚只读「模型」status chip(指向运维管理)。

角色降级与既有范式

  • 读者视角 = 仅「交付通道」区(MCP 卡隐藏管理注记、日报区整体不渲染、模型 chip 不显示);管理员全量。
  • RAG 关闭时:两个语义工具的「RAG」章翻为 stamp-idle「RAG 未启用」,行文字灰显——不隐藏,能力可见但不可用。
  • 复用:.stamp/.mini-seg/检视器 hairline 三段分区/ledger-pipeline 步进条 (替代 DailyBriefFlow 动画编排——feedback-only:进行中段实底 + 计数走字,不再做流动动效); 代码块/端点行 = well 底被动容器。
  • 流式页(文档型),不用全屏仪表壳;CTA 预算 = 唯一 primary「立即生成日报」。
