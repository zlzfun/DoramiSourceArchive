# 哆啦美登录页 · 运镜与演出剧本(自然语言完整描述)

> 本文是现行登录页(v3.13「登录电影化」)的纯自然语言描述,用于「照剧本重拍」:
> 实现者只依据本文,不参考任何既有代码,从零生成一版复刻。

## 一、整体风格

一座**深空剧场**。全屏近黑的舞台(底色是带一丝蓝的黑,约 `#04040e`),右半侧朝登录卡方向有一团**极淡的靛蓝辉光**(品牌靛 `#5b54e8` 系),整个画面覆盖一层细腻的**胶片噪点**(overlay 混合、半透明),四周有**暗角**把视线收向中央。文字主色是近白的淡蓝紫。中文大标题用**衬线字体**,小标签与 eyebrow 用**等宽字体**加大字距,正文用无衬线。气质:安静、贵重、有仪式感的科技电影开场——不是炫技的粒子秀,一切运动都克制且会停下来。

## 二、背景:信号星座(三维摄像机点云)

背景是一张全屏 Canvas,里面漂浮着一片**三维点云星座**:

- **点**:数量随视口面积在约 90–190 颗之间取值。每颗点有三维坐标:x/y 铺满约 0.95 倍视口宽高的范围,z 在「近平面约 150」到「远平面约 1650」之间随机。半径基数约 0.8–2.2 像素,颜色是淡靛(`rgba(150,146,248,…)`),透明度随深度衰减:太近(将穿过镜头)与太远都变透明,中景最亮;点带一圈柔和的辉光(shadowBlur 约 6,随投影比例放大)。
- **投影**:标准透视投影,焦距约 560;屏幕坐标 = 中心 + 三维坐标 × (焦距 / z)。z 小于约 50 的点裁掉不画。
- **连线**:任意两点的**屏幕距离**小于约 150px 时画一条 1px 细线,颜色靛蓝(`rgba(123,118,236,…)`),透明度 =(1 − 距离/150)× 约 0.4 × 两端点深度透明度的较小值。整体效果是稀疏的、随镜头运动不断重组的「星座连线」。
- **漂移**:每颗点在自己的 x/y 上叠加慢速正弦漂移(各自随机的频率 0.5–1.8、相位、振幅 10–34px),漂移相位只在镜头运动时推进——镜头停,漂移也停。

**运镜交互**(这是背景的灵魂):

- 鼠标**横向移动** → 摄像机**偏航**(绕场景中心垂直轴转头),移动量乘以很小的系数注入偏航角速度,上限约 ±0.55 rad/s;
- 鼠标**纵向移动** → 摄像机**推轨**(dolly):上移是推进、下移是后拉,注入 z 向速度,上限约 ±780/s;点穿过近平面就回绕到远平面并随机换个横纵位置(反之亦然),形成无限深度;
- 两种速度每帧乘以约 0.9 的**摩擦系数**衰减;衰减到阈值以下就归零,**两者都为零时立刻停掉 requestAnimationFrame**——手停画面冻结,零后台开销;
- 鼠标再动即唤醒。`prefers-reduced-motion` 时完全不监听鼠标,只画静止的一帧。
- 窗口 resize 重建点云并重画;devicePixelRatio 封顶 2。

**开场落定**:承载背景的容器初始被放大到 1.05 倍,进入第二幕时用约 2.6 秒的缓出曲线缩回 1 倍——像镜头「落稳」。

## 三、布局

内容层是一个居中的双栏网格,最大宽约 1180px,左右比例约 1.05 : 0.95,列距很宽(40–120px 自适应)。**左栏是题词区(hero)**,**右栏是登录卡**。移动端单列堆叠。

### 左栏(hero)自上而下:

1. **品牌行**:72px 圆角方形 logo(产品吉祥物图;图挂了则退化为渐变靛底 + 图标),logo 背后有一圈 **conic 极光光环**——靛紫→天蓝的锥形渐变,模糊约 7px,缓慢旋转(约 6s/圈)并以约 3.4s 周期呼吸明暗;右侧是衬线字 wordmark「DORAMI」,27px、字距 0.14em、近白。
2. **eyebrow**:等宽小字「AI 资讯 · 聚合 · 订阅 · 检索」,12px、字距 0.34em、亮靛。
3. **主标题**:衬线,42–72px 自适应,行高 1.1,两行:「让 AI 资讯」/「有处可栖。」,第二行整行是**靛蓝渐变文字**(亮靛→品牌靛,带柔和投影光)。标题底下有极淡的靛色文字光晕。
4. **lede**:「汇聚多源 AI 资讯,订阅、检索、随手可读——尽在一处。」15px、行高 1.8、灰蓝。
5. **功能徽标**:三枚胶囊——雷达图标「多源聚合」、档案图标「全文留存」、星光图标「语义检索」。淡靛描边、极淡靛底、轻微背景模糊,13px 粗体。

### 右栏(登录卡):

**玻璃拟态浮卡**,圆角大(约 16px),内边距 30–44px:

- 底是 165° 的深蓝半透明渐变(上亮下暗),`backdrop-filter: blur(26px) saturate(1.3)`;
- 1px 淡靛描边;卡顶有一条 1px 的**高光渐变线**(两端透明中间亮);
- 右上角卡内有一团径向靛晕(blur 约 50px);
- 多层阴影:内侧上白下黑各 1px、外侧 40px/90px 的深色投影 + 60px 的靛色氛围光;
- **描边流光**:一束亮点(约 60° 宽的 conic 亮段,靛紫)沿圆角边框**匀速环绕**,约 7 秒一圈——用 conic 渐变旋转角度 + 遮罩只露出 1px 边缘实现。

卡内(表单,纵向排列):

1. mono eyebrow「ACCOUNT ACCESS」(11px、字距 0.3em、亮靛);
2. 标题「欢迎回来」28px 极粗;副题「登录账号,继续浏览你的 AI 资讯。」13.5px 灰蓝;
3. 两个字段:标签(「账号」「密码」12.5px)+ 输入条:高 50px、圆角、深色半透明底、内阴影、左侧一枚 18px 图标(人形/钥匙);**聚焦时**边框变亮靛、外圈浮出 4px 靛晕、再加一层向下的靛色投影,图标同步变亮;placeholder「输入登录账号/密码」。
4. **错误槽**:固定高 26px(有无错误不跳版),错误文案 13px 粉红。
5. **提交按钮**:高 52px、圆角、135° 靛蓝渐变底、白字 15.5px 极粗、内沿 1px 白高光、大片靛色投影;hover 上浮 2px、提亮 8%、右侧箭头图标右移 3px;按下回落;提交中禁用、转圈图标 +「正在登录」。按钮上还有一束**白色高光斜掠**(60% 宽、-18° 斜切),定期扫过一次,hover 立即再扫。
6. 脚注:「受 HMAC 会话保护 · 哆啦美」11px 居中灰蓝。

底部屏幕中央(仅第一幕)有一行提示:「按 **任意键** 进入」,12px、呼吸闪烁。

## 四、三幕演出

**第一幕「标题卡」(0 – 3.6s,任意点击/按键立即跳到第二幕):**

- hero 整体被 transform 平移到画面中心并放大 1.2 倍(**始终左对齐**,只用位移居中——这样两幕之间不会有对齐跳变);
- 品牌行、eyebrow 以「上浮 26px + 模糊 6px + 微缩 0.985 → 清晰」的方式错峰入场(延迟约 120ms / 260ms,时长约 1.1s);
- 主标题**逐词浮现**:「让 」「AI 」「资讯」「有处可栖。」四段,延迟依次约 560 / 750 / 940 / 1130ms,每段从「下沉半个字高 + 顺时针微旋 1.5° + 模糊 8px」升起转清,1200ms、强缓出弹性曲线;
- lede 在约 900ms 后同样上浮入场;
- 功能徽标与登录卡**整幕隐藏**:徽标下沉 16px 全透明;登录卡右移 70px、缩 0.9、模糊 14px、全透明、不可交互;
- 屏底「按任意键进入」提示呼吸。

**第二幕「登录卡浮现」(第一幕结束时触发):**

- hero 用约 1.5s 的弹性缓出**归位**到左栏原位、恢复 1 倍;
- 背景容器同时完成 1.05 → 1 的「落定」缩放;
- 功能徽标延迟约 450ms 后上浮亮起(约 1.1s);
- 登录卡在一个 1300px 透视的容器里,用约 1.3–1.5s 从「右移 + 缩小 + 模糊」浮现归位;
- 屏底提示淡出。

**第三幕「品牌变幻」(第二幕稳定后再等约 2.8s):**

- 原主标题(「让 AI 资讯 / 有处可栖。」)整体**上抬 10px + 模糊 6px 淡出**,约 640ms;
- 同一位置叠放的品牌名两行——「哆啦美」(亮靛 accent,带光晕)/「AI 资讯平台」——**逐字**自下 16px、模糊 12px 缓升转清,每字错峰约 55ms,单字约 1s;同时整块字距从 0.24em **收拢**到 0.02em(约 1.5s),像铅字排紧;
- 无障碍:标题的可读语义始终是完整的一句(aria-label),两层视觉互换均 aria-hidden。

**Reduced motion**:三幕全部跳过,直接呈现第二幕完成态 + 品牌名成形态;背景静止一帧;所有循环动画(光环旋转、流光、gloss)停用。

## 五、彩蛋与杂项

- **蒙眼彩蛋**:密码框聚焦时,logo 上交叉淡入一张「蒙住眼睛」的贴图(约 320ms);失焦淡出。贴图在基础 logo 加载完成后、浏览器空闲时预取,保证聚焦瞬间不加载延迟。
- 错误提交:错误文案出现在固定高度的错误槽,不引起布局跳动。
- 登录请求失败显示后端错误消息,默认「登录失败」。
