# 前端视觉打磨路线（书面化）

> 性质：**视觉/信息设计打磨路线**（分阶段执行计划），非架构重构。与 `frontend-refactor-plan.md`（组件工程线，已在 `refactor/frontend` 分支完成阶段 0-4）互补，聚焦「用户视角的美观、布局、一致性、优雅度」。
> 来源：2026-07-03 一轮全页面截图目检（10 张，浅色模式 · admin 视角，覆盖知识台账 / 节点管理×2 / 采集范围 / 任务与运行 / 接入集成×2 / 运维管理×3），逐项经代码核实带 `file:line` 锚点。
> 前置：建议在 `refactor/frontend` 合并后执行（或直接基于该分支续做），避免两条线冲突。
> 覆盖缺口：**暗色模式、阅读器（user 角色）、登录页、动效** 本轮无截图，未评估 —— 见「待补评估」。
> 逐项进度追踪：执行启动后在 `frontend-refactor-progress.md` 追加「视觉打磨」章节（同一活文档纪律）。

## 0. 总评与总原则

**总评**：设计系统底子好 —— 品牌紫主色、圆角/阴影/留白节奏、页头结构（大标题 + 副标题 + 右上分段切换）、状态胶囊语汇在全部页面高度一致，已是完整产品质感。拉低观感的不是风格不统一，而是**「原始数据裸奔」与「色彩语义失控」**两类局部问题。

1. **不动布局骨架、不动设计令牌体系**。全部改动都在「内容呈现层」：摘要清洗、空值收敛、配色降噪、按钮排布、文案拉平。令牌/角色类唯一事实来源仍是 `frontend/src/index.css`，改样式前对照 `docs/frontend/conventions.md`。
2. **信息噪声优先于装饰优化**。先消灭"看起来像 bug"的裸 markdown / 裸 JSON / 空占位符，再做配色与图表精修，最后才是锦上添花。
3. **语义色只留给语义**。红=危险、绿=健康、琥珀=注意；类别区分一律降级到图标/细条/图例，不再染大号数字。既有「图表数字 hover 才显」约定保留，但**排行榜类图表例外**（榜单本来就是看数的）。
4. **每阶段验收** = `npm run lint`（零警告保持）+ `npm run build`（零警告保持）+ 改动页面截图前后对比；涉及暗色路径的项须双主题目检（bridge 桥接段未退役，暗色是回归风险区）。

## 阶段 A · 数据裸奔清理（P0，半天级）

**目标**：消灭全部"看起来像 bug"的原始数据直渲染。观感提升最大的一步。

1. **摘要剥离 Markdown**（V1）：`utils/readerText.js` 新增 `stripMarkdown(text)` —— 去 `#` 标题符、`![](…)` 图片、`[文本](url)` 保留文本、`*`/`` ` `` 强调符、裸 URL、多余空白折叠。`DataTab.jsx:561` 摘要行套用；ReaderTab 列表摘要（`excerptOf`）同源复用。纯前端展示层清洗，不动存储。
2. **空值不渲染**（V3）：
   - `FetchRunsTab.jsx:547-552`：节点卡的 cron 框仅在「该节点有 cron 覆盖或任务有整体 cron」时渲染；参数框仅在 `Object.keys(params).length > 0` 时渲染。无覆盖节点只剩名称行，卡片自然收缩（预期任务页滚动长度 −2/3）。
   - 有参数时不再 `JSON.stringify` 裸串，翻译成键值 chips（`条数 10`、`抓详情 ✓`）；键名映射复用 fetcher `get_parameter_schema()` 已下发的 `label`（`fetchersById` 前端已有）。`FetchTab.jsx:940-941` 同改。
   - `FetchTab.jsx:913`："无说明"占位删除，无 description 时只显 `N 个节点`。
3. **枚举→中文标签**（V5）：`DataTab.jsx:536` 类型胶囊经 `CONTENT_TYPE_LABELS` 常量映射（`web_article`→资讯文章、`arxiv`→学术论文、`github_release`→版本发布…未命中回落原值）。常量放 `utils/`，后续阅读器/筛选下拉共用。

**验收**：台账页摘要无任何 markdown 符号；任务展开态无 `{}`/`cron：-` 空框；类型胶囊全中文。lint/build 零警告保持。

## 阶段 B · 色彩语义收敛（P0，半天级）

**目标**：KPI 与图表回归「语义色只留给语义」。

1. **KPI 数字去彩虹**（V2）：`adminUtils.js:4` `KPI_COLOR` 全部指标数字回归中性强色（`text-slate-900` / 既有 `.stat-number` 默认），类别色移到 StatCard 左侧图标（icon 着类别色，数字不染）。特例：
   - 向量化率（`AdminOpsTab.jsx:691`）改**阈值语义**：≥80% 中性、40-80% 琥珀、<40% rose —— 0% 再不会是绿的。
   - 收藏总数（`:690`）rose → 中性（icon 可留 Heart rose，数字不染红）。
2. **TOP 榜单重做**（V8）：文章收藏 TOP 改单一品牌色条 + **行尾直标数值**（榜单例外于 hover-only 约定）；条长按榜内最大值归一。若收藏数区分度仍低（全是 1-2），退化为带数字徽标的排行列表（无条）。
3. **各源三指标条图降噪**（随手）：灰色轨道底改更淡或去除，三系列条间距收紧；图例保持。

**验收**：内容/用户/AI 三个子页 KPI 数字全中性、无语义碰撞；TOP 榜一眼可读出数值排序。双主题目检（涉及 `stat-number` 与图表 token）。

## 阶段 C · 交互语义修正（P1，半天级）

**目标**：按钮排布、状态呈现、控件语言三类"交互层面的不优雅"。

1. **删除按钮隔离**（V4）：`FetchRunsTab.jsx:519-521` 与 `FetchTab.jsx:927` 一带的操作行，删除移至行尾 `ml-auto` 分离（与主操作簇拉开视觉距离）；保持既有 `useConfirm` 二次确认不变。不引入溢出菜单（改动最小原则）。
2. **MCP 状态单一化**（V6）：`MCPTab.jsx` 保留 hero 右上一处状态胶囊为唯一状态源；`:291` 的重复胶囊删除。MCP 停止时 hero 文案条件切换（"MCP 服务未启动，启动后即可接入"），令牌区保留但降低视觉权重（透明度/去主按钮），红色告警条保留（它承载"联系管理员"的行动指引）。
3. **日报表单控件统一**（V7）：`DailyBriefPanel.jsx:263-264` 原生 checkbox 换全站滑动 toggle（复用运维页「用户 AI 功能」同款结构/类）；`:271/283` 双「保存」合并为配置区尾部单个保存（一次提交 cron + 精选条数，两个 API 调用并发即可，接口不动）。

**验收**：三处改动页面截图对比；日报 toggle 与运维页 toggle 同观感；MCP 页停止态无矛盾文案。

## 阶段 D · 锦上添花（P2，随手/机会式）

不设独立里程碑，触碰对应文件时顺带：

1. **台账双日期列合并**（v1）：主显「原始发布日期」，抓取时间降为同格第二行小字（或 title 提示），释放宽度给标题列。

## 暂缓 / 不做（本轮明确不动）

- **不重排任何页面布局骨架**（页头/分段切换/卡片栅格保持原样）—— 一致性是现状最大资产。
- **不引入新图表类型/图表库配置大改** —— 只调色与标数。
- **不做 admin 面 fetcher id 的隐藏**（`web_anthropic_news` 等对管理员是有效调试信息，保留）。
- **暗色/阅读器/登录/动效**相关改动 —— 材料未备，见上表。
